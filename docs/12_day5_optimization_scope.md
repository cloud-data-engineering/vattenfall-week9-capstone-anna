# Day 5 optimization scope

Day 5 is the final technical hardening day of the Vattenfall capstone.

It does not create a new pipeline layer. Instead, it reviews, debugs, optimizes, validates, and prepares the completed project for presentation.

## Purpose

Day 5 focuses on making the project:

- easier to understand
- easier to debug
- easier to validate
- more efficient where reasonable
- ready to present

## Main work

Day 5 includes:

- full pipeline review
- final table and view inventory
- performance issue scan
- query plan inspection with `explain(True)`
- debugging scenario documentation
- targeted optimization improvements
- final validation
- presentation readiness

## Engineering principle

The goal is not blind optimization. The goal is to inspect first, identify real risks, apply targeted improvements, validate the result, and document the reasoning.
