# Day 5 debugging notes

Day 5 documents debugging scenarios as reusable engineering knowledge.

## Debugging approach

Each debugging scenario includes:

- symptom
- likely cause
- investigation steps
- fix
- prevention
- final result

## Scenario categories

Relevant debugging categories include:

- path errors
- table name errors
- schema mismatches
- missing columns
- duplicate grains
- join mismatches
- empty outputs
- invalid metric values

## Principle

Good debugging starts by classifying the problem before changing code.
